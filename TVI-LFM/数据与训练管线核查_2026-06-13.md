# 数据与训练管线核查（2026-06-13）

## 结论摘要

当前数据规模不是“小到无法训练”的级别，但跨模态 RGB-IR 任务本身比之前的 RGB-only / IR-only same-modality 结果难很多。此前 80%+ 的结果主要来自 same-modality，不应直接和 RGB-IR cross-modality 的 15%-25% Rank-1 对比。

本次发现一个明确的数据错误：原始 `vcm_train_tracklets.json` 中有 39 条 VCM tracklet 的帧范围混入了不同 pid / modality / camera 的帧，其中 17 条混入了不同 pid，19 条混入了不同 modality，39 条混入了不同 camera。已修复 prepare 和 Dataset 读取层，重新生成 VCM train 索引后问题数为 0。

2026-06-13 更新：LLCM 因图像质量和泛化收益不足，已从当前外部预训练主线移除。后续默认使用干净的 VCM-only RGB-IR tracklet 预训练，再进行 SYSU-MM01 fine-tune。报告中 LLCM 相关统计仅作为历史核查记录，不再作为推荐实验路线。

## 已修复的实质性错误

### 1. VCM prepare 阶段只信 source range，未过滤异常帧

问题：

部分 `track_train_info.txt` 的 source range 起点对应的 `train_name.txt` 帧属于前一个身份/模态/相机。例如某条声明为 `pid=0004, rgb, D2` 的 tracklet，range 开头实际是 `0003/ir/D6`。

修复：

- `tools/prepare_vcm_safe.py`
  - 新增 frame metadata 解析。
  - 根据 tracklet 声明的 `pid/camid/modality` 过滤帧。
  - 重新生成 `vcm_train_tracklets.json`。

### 2. Dataset 读取旧 VCM index 时没有二次防御

修复：

- `project/sysumm01/datasets/vcm.py`
  - VCMTrackletDataset 加载时过滤不匹配帧。
  - SYSU IR + VCM IR 混合 Dataset 加载时过滤不匹配帧。

- `project/sysumm01/datasets/external_rgb_ir.py`
  - External VCM tracklet 加载时过滤不匹配帧。

修复后复查：

- full VCM index 问题 tracklet：0
- SCHP filtered VCM IR index 问题 tracklet：0
- ExternalRGBIRDataset 中 VCM 问题 sample：0

### 3. VCM+LLCM 配置名与真实采样不一致（历史问题，已改为移除 LLCM）

问题：

`external_pretrain_vcm_llcm.yaml` 之前写着 VCM+LLCM，但实际：

```yaml
vcm_ratio: 0.0
```

处理：

- 删除 `external_pretrain_llcm_only.yaml`
- 删除 `external_pretrain_vcm_llcm.yaml`
- 删除 `sysu_finetune_from_llcm_only.yaml`
- 删除 `sysu_finetune_from_vcm_llcm.yaml`
- 删除生成索引 `data/external_indices/llcm_train_images.json`
- 新增 `external_pretrain_vcm_only.yaml`
- 新增 `sysu_finetune_from_vcm_only.yaml`

### 4. 跨模态配置默认包含 val IDs

问题：

多个 `rgb_ir_*.yaml` 默认 `use_val: true`，会导致与当前干净协议不可比。

修复：

- 跨模态配置统一改为 `use_val: false`。
- 显式设置：

```yaml
selection_split: val
final_split: test
```

## 仍需注意但未作为错误修改

1. 当前数据量足以做 baseline，但对比顶会级跨模态 ReID 仍偏小，且 SYSU 本身 train IDs 只有 296。
2. 当前 loss 可以让训练 loss 明显下降，但不保证 hard positive / hard negative 排序改善。
3. 旧 80%+ 结果是 same-modality，不是 RGB-IR cross-modality。
4. 旧 cross-modality 结果中部分配置曾使用 `use_val=true`，需要按干净协议复跑才能公平比较。
5. evaluator 的 camera filtering 规则会影响 Rank-5/10/20 的解释，但对当前 Rank-1 低的问题不是主因。实测当前规则与只移除同 pid 同 camera 的规则 Rank-1 基本一致。

## 当前判断

当前确实存在过数据错误，主要在 VCM index。这个错误已修复，但它只影响 39 / 11061 条 tracklet，不足以单独解释所有跨模态指标偏低。

更大的问题仍然是：单模态特征强，但跨模态共享空间和 hard ranking 没有真正建立。修复数据并移除 LLCM 后，建议先重跑干净的 VCM-only RGB-IR tracklet pretrain，再做 SYSU fine-tune，避免继续基于旧错误 checkpoint 判断方法有效性。
