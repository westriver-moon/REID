# TVI-LFM 低指标根因排查与验证

生成日期：2026-06-13  
目标：解释当前 SYSU-MM01 RGB-IR 跨模态指标显著偏低的根本原因，并用现有日志、checkpoint 和新增诊断实验进行验证。

## 一、结论先行

当前低指标的主因不是 backbone 完全无效，也不是 SYSU 数据泄露或评估脚本明显错误，而是：

```text
单模态身份表征已经能学到；
但 RGB/IR 进入同一 embedding 空间后，真实 gallery 排序中的 hard positive / hard negative 没有被解决。
```

更具体地说：

1. 模型能改善平均跨模态距离，但不能把困难正样本排到困难负样本前面。
2. 外部数据的直接迁移能力很弱，VCM 当前又主要只作为 IR-only 补充，无法提供完整 RGB-IR 配对监督。
3. 当前 adapter、shared head、prototype、contrast 等方法大多提升平均距离或中心对齐，对局部邻域排序帮助有限。
4. 单模态能力明显强于跨模态能力，说明瓶颈集中在跨模态对齐，而不是基础身份识别能力。
5. 当前 best epoch 选择、低 trial 监控和 final eval 之间存在噪声，但这不是导致 20% 级 Rank-1 的根因。

## 二、已完成的验证实验

### 实验 V1：同一模型的 same-modality 与 cross-modality 对比

使用当前外部预训练后 fine-tune 最好模型：

```text
logs/sysu_external_pretrain/vcm_llcm_partial_shared_finetune_ep20/checkpoints/best.pth
```

额外执行：

```text
same_modality RGB eval
same_modality IR eval
cross_modality eval
```

结果：

| 评估协议 | All mAP | All Rank-1 | Rank-5 | Rank-10 | Rank-20 |
|---|---:|---:|---:|---:|---:|
| same RGB | 69.89 | 73.99 | 92.80 | 96.18 | 98.10 |
| same IR | 70.09 | 57.99 | 84.99 | 93.44 | 98.37 |
| cross RGB-IR | 26.84 | 23.31 | 50.46 | 66.72 | 85.12 |

验证结论：

- 同一模型在 same-modality 下仍有可用身份判别能力。
- cross-modality 大幅下降，说明主要矛盾是 RGB/IR embedding 空间对齐失败。
- 这排除了“模型完全没学会 ReID”作为主因。

### 实验 V2：跨模态正样本排名与 hard pair 诊断

新增诊断脚本：

```text
tools/diagnose_sysu_cross_modal.py
```

输出文件：

```text
logs/sysu_external_pretrain/vcm_llcm_partial_shared_finetune_ep20/diagnostics/sysu_cross_modal_geometry.json
logs/sysu_rgb_ir_dual/B4_h192_sampler_4rgb4ir_ep30/diagnostics/sysu_cross_modal_geometry.json
```

诊断指标含义：

| 指标 | 含义 |
|---|---|
| positive_rank_median | 每个 IR query 第一个正确 RGB gallery 的中位排名 |
| positive_rank_p90 | 90% query 的第一个正确样本排名上界 |
| top10_purity | Top-10 gallery 中同 ID 样本比例 |
| hard_gap | hard positive similarity - hard negative similarity |
| mean_gap | average positive similarity - average negative similarity |
| modality_centroid_acc | 用 RGB/IR 中心区分模态的准确率，越高说明模态残留越明显 |

All 模式结果：

| 实验 | All R1 | positive rank median | positive rank p90 | Top-10 purity | mean gap | hard gap | modality centroid acc |
|---|---:|---:|---:|---:|---:|---:|---:|
| E2 external fine-tune | 23.31 | 7.30 | 47.28 | 11.46 | 0.408 | -0.261 | 61.75 |
| B4 h192 sampler | 21.21 | 7.90 | 52.20 | 11.24 | 0.323 | -0.267 | 63.21 |

Indoor 模式结果：

| 实验 | Indoor R1 | positive rank median | positive rank p90 | Top-10 purity | mean gap | hard gap | modality centroid acc |
|---|---:|---:|---:|---:|---:|---:|---:|
| E2 external fine-tune | 26.79 | 4.00 | 18.43 | 10.22 | 0.425 | -0.169 | 62.56 |
| B4 h192 sampler | 27.52 | 4.10 | 22.33 | 9.56 | 0.330 | -0.172 | 64.47 |

验证结论：

- `mean gap` 为正，说明模型确实把平均正样本拉近、平均负样本推远。
- `hard gap` 仍为负，说明最难正样本仍然比最难负样本更远。
- positive rank median 约 7-8，p90 可到 47-52，说明大量 query 的正确 RGB 并没有进入前排。
- Top-10 purity 只有约 10%-11%，说明局部邻域还没有形成稳定的身份邻域。
- 这直接解释了为什么 mAP/Rank-5/Rank-10 有提升，但 Rank-1 很难上去。

### 实验 V3：数据源与 VCM 可用模态检查

数据统计：

| 项目 | 结果 |
|---|---:|
| SYSU train IDs | 296 |
| SYSU val IDs | 99 |
| SYSU train+val IDs | 395 |
| SYSU test IDs | 96 |
| train+val 与 test ID 交集 | 0 |
| LLCM train samples | 30,921 |
| LLCM pids | 713 |
| LLCM RGB samples | 16,946 |
| LLCM IR samples | 13,975 |
| VCM SCHP filtered tracklets | 4,954 |
| VCM SCHP filtered pids | 500 |
| VCM SCHP filtered modality | IR only |

VCM 原始索引与本地文件存在性快速检查：

| VCM tracklet 类型 | 原始索引数量 | 本地存在情况 |
|---|---:|---|
| RGB tracklets | 6,142 | 约 6,125 个缺失 |
| IR tracklets | 4,919 | 约 4,917 个存在 |

验证结论：

- SYSU train/test 没有 ID 泄露，低指标不是因为 split 错乱或泄露。
- LLCM 是当前真正提供 RGB-IR 配对的外部数据源。
- VCM 在当前实际训练中主要是 IR 域增强，不是完整 RGB-IR 配对训练源。
- 因此，VCM 能帮助 IR 表征，但很难单独解决 RGB-IR 对齐。

### 实验 V4：外部预训练直接迁移 vs SYSU fine-tune

| 实验 | All mAP | All R1 | Indoor mAP | Indoor R1 |
|---|---:|---:|---:|---:|
| LLCM only external pretrain 直接评估 | 7.53 | 5.13 | 13.44 | 5.75 |
| VCM + LLCM external pretrain 直接评估 | 8.56 | 5.86 | 17.16 | 8.42 |
| LLCM only + SYSU fine-tune | 24.76 | 21.83 | 37.42 | 24.50 |
| VCM + LLCM + SYSU fine-tune | 26.84 | 23.31 | 39.68 | 26.79 |

验证结论：

- 外部预训练直接迁移到 SYSU 很弱，域差异明显。
- 加 VCM IR 后有稳定收益，但幅度有限。
- SYSU fine-tune 是必须的；外部数据目前更像初始化/正则化，而不是直接解决跨模态检索。

### 实验 V5：训练历史中的平均 gap 与最终排序矛盾

代表性训练历史：

| 实验 | epoch | train cm gap | all mAP | all R1 |
|---|---:|---:|---:|---:|
| E2 external fine-tune | 1 | 0.049 | 8.86 | 5.72 |
| E2 external fine-tune | 11 | 0.521 | 25.61 | 22.57 |
| E2 external fine-tune | 20 | 0.566 | 27.65 | 23.81 |
| B4 h192 sampler | 1 | 0.024 | 7.50 | 3.71 |
| B4 h192 sampler | 16 | 0.669 | 25.88 | 21.13 |
| B4 h192 sampler | 30 | 0.708 | 24.68 | 20.19 |

验证结论：

- batch 内平均 `cm_gap` 增大确实能带来前期收益。
- 但 `cm_gap` 继续变大后，B4 的 Rank-1 反而回落。
- 结合 V2 的 hard gap 为负，说明平均距离不是充分指标；真正需要优化真实检索排序中的 hard pair。

## 三、根因清单

### 根因 1：hard positive / hard negative 排序没有解决

状态：已验证。

证据：

- E2 All hard gap = -0.261。
- B4 All hard gap = -0.267。
- 正样本 median rank 仍在 7-8，p90 可到 47-52。

解释：

当前 loss 可以让同 ID 平均更近、不同 ID 平均更远，但 Rank-1 由最难正样本和最难负样本决定。只要 hard negative 排在 hard positive 前面，Rank-1 就上不去。

### 根因 2：VCM 当前不是完整 RGB-IR 配对监督源

状态：已验证。

证据：

- SCHP filtered VCM tracklet 全部是 IR。
- VCM RGB 大部分本地文件已不存在。
- external pretrain 日志显示：`vcm: rgb=0, ir=4936`。

解释：

VCM 对 IR 侧有帮助，但不能提供同 ID RGB/IR pair 来学习跨模态映射。因此它更适合提升 IR backbone 或 IR 数据覆盖，而不是直接承担 RGB-IR 对齐。

### 根因 3：外部数据与 SYSU 测试分布差异较大

状态：已验证。

证据：

- VCM+LLCM external pretrain 直接 SYSU cross eval 只有 All mAP 8.56 / R1 5.86。
- SYSU fine-tune 后提升到 All mAP 26.84 / R1 23.31。

解释：

外部数据不能直接泛化到 SYSU。当前 external pretrain 主要是“提供初始表征”，不能替代 SYSU 上的跨模态适配。

### 根因 4：单模态能力强，跨模态共享空间弱

状态：已验证。

证据：

- 同一 E2 checkpoint same RGB Rank-1 73.99。
- same IR Rank-1 57.99。
- cross RGB-IR Rank-1 23.31。

解释：

模型不是不会做 ReID，而是 RGB 特征和 IR 特征还没有进入一个足够统一、可检索的几何空间。

### 根因 5：adapter 容量不是主要瓶颈

状态：由已有消融支持。

证据：

| 实验 | All mAP | All R1 |
|---|---:|---:|
| no adapter | 24.75 | 20.61 |
| h64 | 24.75 | 20.73 |
| h192 | 25.46 | 21.28 |
| h384 | 25.36 | 21.28 |

解释：

adapter 从无到 h192 有小幅收益，但 h384 不继续提升，说明简单扩大后处理 MLP 容量不能根治跨模态 hard ranking。

### 根因 6：shared Transformer head 也没有自动解决对齐

状态：由已有消融支持。

证据：

| 实验 | All mAP | All R1 |
|---|---:|---:|
| B4 h192 sampler | 25.70 | 21.21 |
| D1 2-layer shared head | 24.92 | 21.43 |
| D2 4-layer shared head | 23.60 | 19.81 |

解释：

增加 token-level shared head 并没有显著改善。4-layer 反而退化，说明问题不是“头不够深”，而是对齐目标和训练数据不足以约束真实跨模态邻域。

### 根因 7：模态信息仍残留在 embedding 中

状态：已部分验证。

证据：

- E2 modality centroid acc：All 61.75，Indoor 62.56。
- B4 modality centroid acc：All 63.21，Indoor 64.47。

解释：

用简单 RGB/IR 中心仍能以约 62%-64% 准确率区分模态，说明 embedding 没有完全模态不变。但这不是唯一原因，因为 acc 不是极高；它更像辅助症状。

### 根因 8：prototype / center alignment 只能对齐中心，不能压紧困难样本

状态：已部分验证。

证据：

| 实验 | prototype distance | cross-modal intra radius | hard gap |
|---|---:|---:|---:|
| E2 All | 0.796 | 0.464 | -0.261 |
| B4 All | 1.018 | 0.491 | -0.267 |

解释：

类中心距离并不能代表每个样本都正确混合。即使中心有所靠近，类内半径和 hard pair 仍可能导致错误排序。

### 根因 9：checkpoint 选择存在低 trial 噪声，但不是主因

状态：已验证为次要问题。

证据：

- 训练中每 epoch 多数使用 2-trial 监控，final 是 10-trial。
- B4 best epoch 约 13，后期训练 loss 继续下降但 cross R1 回落。

解释：

低 trial 会影响选点稳定性，后期也可能轻微过拟合。但这个问题解释不了 same-modality 与 cross-modality 的巨大差距，因此不是根本原因。

### 根因 10：多分支 patch / LAST / SCHP 融合不是当前跨模态瓶颈

状态：由已有消融支持。

证据：

- patch 消融多 seed final 均值中，288x144 单分支略高于多分支。
- SCHP feature fusion 未带来稳定提升，SCHP sampling 对 IR 有小收益。
- LAST 可视化效果不稳定，后续已退回 patch_mean 主线。

解释：

局部/patch 结构可以影响单模态表征，但当前最大断点是跨模态 hard ranking。继续堆局部结构不一定能解决 RGB-IR 排序错位。

## 四、目前不是主因的项目

| 怀疑项 | 判断 | 证据 |
|---|---|---|
| SYSU train/test 数据泄露 | 不是主因 | train+val 与 test ID 交集为 0 |
| RGB 误加载 IR checkpoint | B4 中不是 | B4 初始化日志显示 RGB/IR 分别加载各自 checkpoint |
| backbone 完全失效 | 不是 | same RGB / same IR 仍显著高于 cross |
| 单纯训练轮次不够 | 不是主要解释 | B4 训练到后期 loss 下降但 R1 回落 |
| 单纯 adapter 参数太少 | 不是 | h192 到 h384 无收益 |

## 五、下一步验证实验建议

下面这些不是“继续堆模块”，而是针对上述根因的验证实验。

| 优先级 | 实验 | 目的 | 预期验证 |
|---:|---|---|---|
| 1 | full-gallery hard mining / hard identity sampler | 直接针对 hard positive/negative | hard gap 转正或 positive rank p90 下降 |
| 2 | 恢复 VCM RGB 后做完整 VCM RGB-IR paired pretrain | 验证 VCM 只做 IR supplement 是否限制对齐 | external direct 与 fine-tune 都应提升 |
| 3 | SYSU camera-aware cross-modal sampler | 让 batch 更接近真实 query-gallery 分布 | Top-10 purity 与 Rank-1 提升 |
| 4 | 用 frozen 单模态 encoder，只训练跨模态 metric head，但 loss 只看 hard rank | 区分 encoder 问题和 loss/sampler 问题 | 如果提升，说明 encoder 足够，目标函数是瓶颈 |
| 5 | 对 top hard IDs 做可视化误检分析 | 判断是否集中在少数身份/摄像头/遮挡 | 如果集中，可做 quality/camera-specific 策略 |

## 七、一句话总结

当前项目的低指标根因可以概括为：

```text
不是模型不会识别人；
而是 RGB/IR 特征虽然平均拉近了，却没有在真实检索 gallery 里解决 hard positive 与 hard negative 的排序。
同时，VCM 当前缺少可用 RGB 配对，使外部数据对跨模态对齐的贡献受限。
```
