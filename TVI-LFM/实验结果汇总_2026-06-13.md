## 一、关键结论

1. 当前保留的有效主线是两条单模态分支：RGB 侧 `mixed_msmt17_sysumm01/rgb_lastvit_ep40_2026-05-24`，Final All mAP 89.73 / Rank-1 92.95；IR 侧 `sysu_ir_vcm_ir/schp_full_s2_steps800_ep40_2026-05-31`，Final All mAP 86.11 / Rank-1 78.07。
2. 2026-06-14 最后一批 VCM+RegDB 跨模态验证中，`contrast + SYSU 40ep` 的 final test 为 All mAP 28.47 / Rank-1 25.21，确实高于 20 epoch，但仍远低于单模态主线。
3. `trainval` 设置会导致训练中 val 指标膨胀。例如 `contrast + SYSU 40ep` 的 val mAP / Rank-1 达到 68.77 / 63.36，但 final test 只有 28.47 / 25.21。后续不再用这类 val 曲线判断真实性能。
4. 双编码器 adapter、shared Transformer head、VCM RGB-IR tracklet、VCM+LLCM、VCM+RegDB 等跨模态路线均未超过可接受阈值，已经从当前主线移出。
5. Patch 消融中，单次实验里 multi-branch 的训练中 best epoch 更高；但三次 seed 的 final 均值上，`C_singlebranch_288x144` 略高于 `B_multibranch_288x144`，说明多分支 patch 的收益还需要更严格地复核。当前单模态主线仍暂保留多分支 patch 作为既有结构。

## 七、当前比较建议

1. 本节更新为历史总结：跨模态路线已停止继续投入，相关实验配置和日志已清理。
2. 当前只保留两条单模态训练资产：RGB 侧 `mixed_msmt17_sysumm01_rgb_lastvit.yaml`，IR 侧 `sysu_ir_vcm_ir_schp_full_s2_steps800.yaml`。
3. 外部跨模态预训练直接迁移到 SYSU 的指标很低，SYSU fine-tune 后也只达到 20%-30% Rank-1 区间，说明当前 RGB-IR 对齐机制不足。
4. Patch 消融还不能强结论地证明 multi-branch 一定优于 single-branch；目前更稳妥的表述是：multi-branch 在部分 seed 的 best epoch 有优势，但 final 多 seed 均值不占优。
5. 后续若重新启动跨模态工作，应先重新定义数据和评估协议，而不是在当前 adapter / shared head / prototype / contrast 组合上继续叠加。
